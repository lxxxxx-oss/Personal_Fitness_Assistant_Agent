# 智能体岗位面试学习中心

这里不是代码说明书，而是一套从零开始的面试课程。目标是让你先理解，再表达，最后能够应对追问。

## 岗位范围与难度梯度

这套资料主要面向校招的“智能体应用开发”和“全栈智能体开发”。公司规模不作限制，因此既准备小团队关心的快速落地，也准备中大型公司关心的可靠性、评测、安全和系统设计。

| 难度 | 常见阶段 | 你要做到什么 |
|---|---|---|
| 基础级 | 一面概念题 | 用普通话解释是什么、解决什么问题 |
| 项目级 | 项目深挖 | 讲出真实调用链、数据结构、取舍和代码证据 |
| 工程级 | 二面或系统设计 | 分析故障、指标、安全、扩容和生产化方案 |

不要跳过基础级直接背工程术语。渐进式学习的目标不是每道题都答得很长，而是面试官继续追问时，每一层都有新信息。

## 每个知识点的讲透标准

每项核心技术至少要能回答九件事：它是什么；解决什么问题；在本项目的入口、执行和输出；采用了什么数据结构或参数；替代方案与选型理由；常见失败模式；当前怎样处理失败；用什么指标证明有效；生产化先补什么。前五项证明你理解技术，后四项证明你有工程意识。

## 学习目标分为四级

### 第一级：能听懂

你能用生活化语言解释：

- 智能体与普通聊天机器人有什么区别。
- 为什么系统需要先判断任务。
- 为什么不同问题要交给不同能力。

对应文档：[零基础项目地图](./01_零基础项目地图.md)。

### 第二级：能讲清

你可以完成：

- 一句话项目定位。
- 30 秒项目介绍。
- 3 分钟完整项目故事。
- 三个核心亮点和个人贡献。

对应文档：[项目讲解与面试话术](./02_项目讲解与面试话术.md)。

### 第三级：能解释

每项核心技术都可以回答：

1. 它是什么。
2. 它解决什么问题。
3. 项目中怎样使用。
4. 有哪些替代方案。
5. 为什么选择当前方案。

对应文档：[简历技术点总表](./03_简历技术点总表.md)和[高频技术问答](./04_高频技术问答.md)。

### 第四级：能防守

面对连续追问时，你能：

- 先给结论，再解释原因。
- 画出数据流和模块关系。
- 说明当前做到哪里。
- 说明生产化如何升级。
- 承认合理边界，但不主动削弱项目价值。

对应文档：[技术深挖与白板](./06_技术深挖与白板.md)和[模拟面试](./08_模拟面试.md)。

## 统一回答结构

所有技术问题尽量使用同一套结构：

```text
结论
  -> 业务问题
  -> 方案怎样工作
  -> 为什么选择它
  -> 结果、边界与下一步
```

例如，面试官问“为什么使用 LangGraph”，可以回答：

> 因为这个项目不是单一问答，而是要在知识检索、联网搜索、动作分析和外部工具之间选择流程。LangGraph 把每一步表示成明确节点，让路由结果、工具结果和错误都保存在共享状态中。相比一个巨大 Prompt，它更容易控制和测试；相比完全自主的多智能体方案，它更适合健康场景对稳定性的要求。

## 技术主线只分三层

| 层次 | 负责什么 | 代表技术 |
|---|---|---|
| 控制层 | 决定任务怎样流转 | LangGraph、StateGraph、Router |
| 能力层 | 完成检索、搜索、计算和工具调用 | RAG、Milvus、Tavily、Motion、ToolRegistry、MCP |
| 保障层 | 管理上下文、失败和质量 | Memory、PromptBuilder、ToolResult、fallback、测试与评测 |

先记住三层关系，再学习具体名词。面试时也应该先讲系统结构，再展开某个技术点。

## 三种表达口径

| 口径 | 什么时候讲 | 示例 |
|---|---|---|
| 价值口径 | 开场和业务问题 | 系统能根据问题选择知识、搜索、算法或工具 |
| 设计口径 | 技术追问 | Router 生成路由计划，LangGraph 调度子工作流 |
| 边界口径 | 深入追问 | 当前是可运行原型，生产化还需权限、监控和数据闭环 |

不要一开场就说“只是原型”，也不要把未来规划说成已经全面上线。先展示核心设计，再在被追问时说明证据和边界。

## 学习检查

如果下面五个问题还不能脱离文档回答，就先不要继续背更多名词：

1. 这个项目为什么算智能体？
2. 用户提出问题后，系统依次做什么？
3. 为什么大模型不能包办所有任务？
4. 路由、工具注册表和 MCP 分别负责什么？
5. 怎样证明系统不是“看起来能用”，而是真的可验证？
