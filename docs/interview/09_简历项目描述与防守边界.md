# 简历项目描述与防守边界

本文只保留当前可使用的简历表述和追问边界。学习概念看[简历技术点总表](./03_简历技术点总表.md)，测试数字看[项目证据](../项目证据.md)；这里不再保存已经淘汰的多版草稿。

## 使用规则

- 简历写已经实现且有证据的主线，不堆所有内部细节。
- 面试先讲价值和方案，再主动说明原型边界。
- 评测数字可能变化，提交简历前从项目证据核对，不在多处维护。
- “可选、默认关闭、待完整验收”的能力不能说成默认生产能力。

## 1. 混合意图路由

### 当前推荐表述

> **构建可评测的混合意图路由**：基于 LangGraph StateGraph 组织五类执行路径，以加权规则和歧义检测稳定处理业务边界，并为低置信表达保留词面样例、可选 embedding/Qwen 兜底；记录路由分数、决策原因和执行计划，通过白名单串联复合任务，并使用常规、困难和语义改写集持续回归。

### 证据与边界

- 已实现加权规则、歧义信号、可解释评分、主/次意图和四种两步白名单组合。
- embedding fallback 有置信度和 Top1-Top2 margin 门控；本地 Qwen classifier 已做 A/B。
- embedding 和 Qwen 默认关闭，因为现有小样本没有证明它们比确定性基线更好。
- 离线满分只说明当前样本没有回归，不代表生产流量泛化完成。

追问时强调：规则保护“用户要求系统做什么”的工具边界，embedding 更擅长主题相似，不能无条件替换规则。

## 2. 3D 运动动作相似度分析

### 当前推荐表述

> **实现可解释的动作相似度分析原型**：使用 MediaPipe 与 OpenCV 将图片、短视频转换为统一 PoseSequence，对 3D 关键点序列进行平移/尺度归一化、FastDTW 时序对齐、余弦和形状差异比较，并输出膝髋角度误差、最大偏差关节/帧及低置信质量提示。

### 证据与边界

- 图片、视频和 `.npz` 已统一到带 schema、坐标空间、置信度和 FPS 的 `PoseSequence`。
- 已有全局相似度、局部关节距离、深蹲膝髋角误差和质量门控。
- “3D”指单目 MediaPipe 的 `x/y/z` 或 world landmarks，不是多摄像头专业动捕。
- 正式标准/典型错误动作集、动作周期切分、阶段规则和教练标签校准尚未完成，因此不能称为专业评分模型。

追问时强调：数值链路可重复、可定位错误；多模态大模型更适合作为后续解释层，而不是替代关键点和时序指标。

## 3. Milvus RAG 检索

### 当前推荐表述

> **搭建可追溯、可评测的 RAG 链路**：使用 Sentence-Transformers 生成中文向量并以 Milvus IVF_FLAT/COSINE 检索，支持版本化 chunk、来源透传、同源旧片段清理和模型维度冲突防护；建立带参考答案的黄金集，并复用真实 Retriever 与生成节点进行 RAGAS 上下文相关性、忠实度和答案相关性评测。

### 证据与边界

- Chat/Diet 复用同一 Retriever，检索来源会进入回答元数据。
- 15 条黄金集中，12 条可回答样例进入三指标均分，3 条知识库外问题单独保留，不混淆“正确拒答”和“检索相关性”。
- 本地 Qwen3-8B 裁判与 BGE embedding 已完成单条兼容性烟测；完整 12 条项目基线尚未记录。
- manifest 是原型级入库追踪；持久化批次账本、失败回滚、重排序和大规模评测没有完成。

追问时按 `query -> Retriever -> contexts -> generate_node -> answer -> RAGAS judge` 解释，三项指标分别定位检索器、生成可信度和端到端回答质量。

## 4. 分层记忆与上下文压缩

### 当前推荐表述

> **实现分层记忆与持久化会话压缩机制**：将 SQLite 完整会话、抽取式 active summary、运行时滑动窗口和长期用户记忆分开管理；通过摘要覆盖边界增量处理消息，组合近期原文、已确认长期记忆和 RAG 证据构造 Prompt，敏感信息先进候选确认，摘要或 Milvus 失败时分别降级到滑动窗口和 SQLite 检索。

### 证据与边界

- 完整 turn 事务写入，HTTP、SSE、WebSocket 共用持久化和 Prompt 构建入口。
- 长期记忆支持显式写入、精确去重、CRUD、候选确认和 SQLite FTS/LIKE 召回。
- Milvus 是可选派生索引；当前通过本地 job 处理接口演示，不是常驻 worker。
- active summary 是确定性原文摘录，按字符而非精确 token 控制；模型摘要、冲突替代、加密授权和多实例治理未完成。

追问时强调：完整历史、当前会话进度和跨会话长期事实是三类不同数据，不能都塞进聊天窗口或都写进向量库。

## 简历提交前检查

1. 四项表述与当前代码状态一致。
2. 测试和评测数字从[项目证据](../项目证据.md)核对。
3. 默认关闭的实验能力明确写成“可选”或只在追问时解释。
4. 没有把代码级闭环写成生产上线，也没有把兼容性烟测写成完整项目基线。
