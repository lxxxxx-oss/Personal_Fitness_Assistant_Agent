# 项目运行与排错

本文档集中维护本地开发、测试、评测、Docker 和端侧联调命令。项目能力和当前进度看 [项目总览.md](./项目总览.md)，接口参数看 [接口说明.md](./接口说明.md)。

MCP 是简历和面试材料中的工具协议亮点。默认 `MCP_SERVER_COMMAND=mock` 用于离线测试和稳定演示；显式配置真实 Server 命令时，系统会走 subprocess + stdio JSON-RPC，并在连接失败时记录原因后回退 mock。

以下命令均默认在当前项目根目录执行。两台电脑的绝对路径可以不同，不要把某台机器的目录写进通用启动命令。

## 1. 快速启动

推荐使用 Python 3.11 的 `fitness-agent` Conda 环境：

```powershell
conda activate fitness-agent
pip install -r requirements.txt
$env:LLM_MOCK="true"
$env:RETRIEVER_BACKEND="memory"
$env:MCP_SERVER_COMMAND="mock"
$env:TAVILY_API_KEY=""
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

如果只做稳定网页演示，建议明确启用演示依赖，避免误把 Mock 结果当成真实外部调用：

```powershell
$env:LLM_MOCK="true"
$env:RETRIEVER_BACKEND="memory"
$env:MCP_SERVER_COMMAND="mock"
$env:TAVILY_API_KEY=""
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

以上 `$env:...` 是 PowerShell 语法；如果当前提示符是 `cmd.exe`，要改用 `set LLM_MOCK=true`、`set MCP_SERVER_COMMAND=mock` 等 `set` 命令。Web UI 和 API 由同一个 FastAPI 进程提供，不需要再启动独立前端。

本地模型运行时保持单进程、单 worker。多个 worker 会分别加载模型，可能重新引入数 GB 内存复制；同一 worker 内的子图共享 tokenizer/model，并发生成在共享模型层串行执行。HTTP/SSE/WebSocket 会把同步 LangGraph 工作放入线程，SSE/WebSocket 还通过 asyncio queue 转发 token，因此推理期间事件循环仍能处理其他 I/O；这不代表模型推理已经并行化。

Windows 桌面环境也可以使用项目脚本：

```powershell
.\scripts\run_backend.cmd
```

脚本默认使用 mock MCP，日志写入：

```text
logs/uvicorn.out.log
logs/uvicorn.err.log
```

## 2. 安装依赖

基础后端、RAG、Router、MCP、`.npz` Motion 分析和测试：

```powershell

```

体验 `/motion/analyze-image`、`/motion/analyze-video` 或 Web UI 图片上传时，还要安装 MediaPipe 与 OpenCV：


说明：

- 建议在 Python 3.11 的 Conda 或 venv 环境安装 Motion 依赖。
- 默认模型路径为 `data/models/pose_landmarker.task`。
- 可通过 `MEDIAPIPE_POSE_MODEL_PATH` 覆盖模型路径。
- `data/models/` 和 `*.task` 被 `.gitignore` 忽略，两台电脑需要分别准备。
- 缺少模型文件时，图片和视频姿态接口返回 503 和明确的缺失提示。

用标准动作视频构建与当前 MediaPipe 视频链路同 schema 的参考 PoseSequence：

```powershell
python scripts/build_motion_reference.py .\standard_squat.mp4 `
  --name squat_standard `
  --library-dir data\motions
```

RAGAS 语义评测：

```powershell
python -m pip install -r requirements-eval.txt
$env:OPENAI_API_KEY="你的裁判模型密钥"
python scripts/eval_rag.py --show-cases
```

本机离线裁判推荐采用“两进程、一个 Conda 环境”的轻量方式：项目与 RAGAS 继续运行在 `fitness-agent` 环境，Qwen3-8B 由 Windows 原生 `llama.cpp` 独立加载，不需要为模型再创建 Python 环境。这样可避开原生 Windows 下 AWQ 后端安装困难，也不会污染项目依赖。

本机已选用官方 `Qwen/Qwen3-8B-GGUF` 的 `Q4_K_M` 量化文件，建议路径为：

```text
D:\Users\Agent\model\models\judges\Qwen\Qwen3-8B-GGUF\Qwen3-8B-Q4_K_M.gguf
```

安装并启动本地裁判服务：

```powershell
winget install --id ggml.llamacpp --exact
llama-server `
  -m "D:\Users\Agent\model\models\judges\Qwen\Qwen3-8B-GGUF\Qwen3-8B-Q4_K_M.gguf" `
  --host 127.0.0.1 --port 8081 `
  -ngl 99 -c 8192 `
  --reasoning off
```

另开一个 Anaconda PowerShell，进入项目环境后运行：

```powershell
conda activate fitness-agent
$env:OPENAI_API_KEY="local-only"
python scripts/eval_rag.py `
  --judge-model Qwen3-8B `
  --judge-base-url http://127.0.0.1:8081/v1 `
  --judge-embedding-provider huggingface `
  --judge-embedding-model "D:\Users\Agent\model\models\embeddings\BAAI\bge-small-zh-v1.5" `
  --show-cases
```

`local-only` 只是满足 OpenAI 客户端的非空字段要求，不是真实密钥；服务只监听 `127.0.0.1`。embedding 参数使用本地绝对路径，避免 Hugging Face 客户端为了确认仓库文件而联网。`--reasoning off` 用于避免 Qwen3 的思考内容干扰 RAGAS 结构化评分。16 GB 显存可优先尝试 `-ngl 99` 全量 GPU offload；若启动时显存不足，先减小 `-c`，再降低 `-ngl`。如果当前终端刚安装 llama.cpp 后还找不到 `llama-server`，关闭并重新打开终端即可刷新 PATH。

`requirements-eval.txt` 将 `langchain-community` 暂时约束在 `0.4.1`：RAGAS 0.4.3 仍会导入一个在 `0.4.2` 中被移除的兼容模块。该约束只服务离线评测环境，不改变项目运行链路；RAGAS 修复上游兼容问题后可再移除。

评测集位于 `data/eval/rag_eval.jsonl`，21 条样例均带人工参考答案。脚本只有一个入口：先使用项目现有 Retriever 检索，再调用项目现有 Chat RAG 生成节点回答，最后由 RAGAS 0.4 的 collections API 输出三项分数：

- `context_relevance`：检索上下文是否与用户问题相关，对应 Retriever 质量，代码直接使用 RAGAS `ContextRelevance`。
- `faithfulness`：生成回答中的事实是否能由检索上下文支持，对应幻觉风险。
- `answer_relevance`：最终回答是否直接回应用户问题，对应端到端有效性。

三项均分只统计 19 条可回答样例。2 条知识库外问题保留在数据集中，但正确拒答属于另一类能力，直接混入上述三项会把“没有相关上下文”误算成检索失败，因此脚本会明确报告其排除数量。当前 12 份可索引知识文档经过分块 v2 得到 81 个 chunk；本地 BGE 在 Top-5、阈值 0.5 的检索冒烟中，对 19 条可回答样例实现正确来源和关键证据 `19/19` 命中。当前代码、单元测试和本地裁判单条中文样例的三指标兼容性烟测已经完成；上述检索冒烟与单样例裁判烟测都不等于项目 Retriever、生成器和 19 条样例的完整 RAGAS 基线，完整运行前不记录或伪造项目语义均分。可用 `RAGAS_JUDGE_MODEL`、`RAGAS_JUDGE_EMBEDDING_MODEL`、`RAGAS_JUDGE_EMBEDDING_PROVIDER` 和 `OPENAI_BASE_URL` 替换兼容服务，密钥只从环境变量读取。云端默认使用 OpenAI embedding；本地 llama.cpp 裁判建议把 provider 设为 `huggingface`，复用本地 BGE，避免依赖 embeddings API。

如果报告中的 Retriever mode 是 `keyword`，说明 Sentence-Transformers 模型没有成功加载；报告仍会如实显示模式，不能把该结果表述成 embedding/Milvus 质量。当前默认模型是 `BAAI/bge-small-zh-v1.5`，新机器需要联网下载或预先准备 Hugging Face 缓存。`LLM_MOCK=true` 时脚本会拒绝运行，因为 mock 答案没有语义评测价值。

默认模型从 text2vec 切换到 512 维 BGE 后曾使用 `fitness_knowledge_bge_small_zh_v15`；结构感知分块 v2 增加章节字段后，当前默认名称进一步调整为 `fitness_knowledge_bge_small_zh_v15_chunk_v2`。旧 Collection 都不会被自动删除。用户如果通过环境变量强制让 BGE 复用旧维度 Collection，检索器会返回 `CONFIG_CONFLICT`，其中包含旧维度、期望维度、模型名和建议的新 Collection 名称，并在允许时降级到内存检索。安全迁移方式是：

1. 保留旧 Collection，便于回滚。
2. 设置新的 `MILVUS_COLLECTION_NAME`，不要直接清空旧 Collection。
3. 重启后端，让启动流程把 `data/knowledge` 重新编码并写入新 Collection。
4. 运行 RAG 黄金集和 Milvus 冒烟测试，确认新索引后再决定是否清理旧数据。

## 6. 局域网与微信小程序联调

以下 `0.0.0.0` 启动方式只用于受信任局域网联调。当前服务没有鉴权、限流或会话所有权校验；不要把端口直接映射到公网。


联调步骤：

1. 微信开发者工具导入 `miniprogram/`。
2. 使用测试 AppID。
3. 开发阶段勾选“不校验合法域名”。

## 8. 配置速查

主要配置位于 `app/config.py`：

| 配置 | 说明 |
|---|---|
| `model_path` | Qwen3 模型路径 |
| `model_device` | `cpu` 或 `cuda` |
| `model_max_tokens` | 最大生成 token |
| `llm_mock` | 是否使用演示 LLM，默认 `false`；设置 `LLM_MOCK=true` 时不加载真实本地回答模型 |
| `llm_router_enabled` | 是否启用本地 Qwen Router A/B，默认 `false` |
| `llm_router_max_tokens` | Router classifier 最大输出 token，默认 128 |
| `router_embedding_enabled` | 是否启用低置信 embedding 样例路由，默认 `false` |

| `router_embedding_min_confidence` | embedding 路由最低置信度，默认 0.68 |

| `embedding_model` | Sentence-Transformer 模型名，默认 `BAAI/bge-small-zh-v1.5`（512 维） |

- `MEDIAPIPE_POSE_MODEL_PATH`

### RAG 分块 v2 与旧 Milvus Collection

当前入库按 Markdown 标题层级、段落和句子分块，普通相邻块也保留约 80 字符重叠；`section_path` 会参与向量编码，并随检索证据进入生成 Prompt。重叠按字符近似，不是精确 token 数。

默认知识 Collection 已改为 `fitness_knowledge_bge_small_zh_v15_chunk_v2`，因此不会删除或覆盖旧 `fitness_knowledge_bge_small_zh_v15`。服务重启后，新 Collection 会按 v2 schema 建立并重新加载知识；同一 `source` 再次入库会先清理它的持久化旧 chunk，避免不同版本并存。

如果环境变量仍显式指向旧 Collection，系统会兼容读写，但旧 schema 没有 `section_path`，检索结果中的章节会为空。希望完整启用章节元数据时，应将 `MILVUS_COLLECTION_NAME` 指向一个新的 Collection 名后重启服务，而不是直接清空旧库。

## 9. 常见运行边界

- Sentence-Transformer 首次加载可能需要联网；失败时检索器降级为关键词匹配。
- Tavily API Key 缺失时 SearchTool 使用 mock 搜索结果。
- `MCP_SERVER_COMMAND` 默认为 `mock`；显式使用真实 MCP Server 时，连接失败会自动降级。
- Motion 图片入口只做单帧静态姿态摘要，不能判断完整节奏、轨迹或发力顺序。
- 本地模型适合单机演示；高并发生产环境应拆成独立模型服务。
