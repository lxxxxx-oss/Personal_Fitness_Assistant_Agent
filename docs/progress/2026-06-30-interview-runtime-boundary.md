# 2026-06-30 面试驱动与可运行边界记录

## 背景

当前项目用于 AI Agent 应用开发方向的面试展示。项目目标不是把所有实现细节打磨到生产级，而是在保证核心链路可运行、可演示、可解释的前提下，形成一套能支撑面试追问的项目表达。

## 本次记录内容

- 在 `AGENTS.md` 中新增“可运行版本与面试表达边界”，作为后续协作的长期约束。
- 在 `docs/interview/INTERVIEW_PREPARATION_OPTIMIZATION_PLAN.md` 中补充同样原则，确保面试资料维护时遵守该边界。

## 后续执行原则

1. 项目必须保持为可以完整运行的版本，核心接口和主流程不能因为规划功能而不可用。
2. 必要功能优先补到可运行原型，非必要功能作为锦上添花的后续规划。
3. 代码层面可以务实取舍，文档层面必须清晰、准确、无低级矛盾。
4. 面试表达可以强调架构预留、原型链路、评测方法和演进路径，但不要把未实现能力写成已完整落地。
5. 后续所有优化都优先服务于面试可信度、现场演示稳定性和追问防守能力。
