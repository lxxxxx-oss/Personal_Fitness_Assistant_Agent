# 本地原型安全与部署边界

## 代码事实

- FastAPI 没有认证或授权中间件，历史接口只按客户端提供的 `user_id` 访问数据。
- CORS 配置为允许任意来源。
- `_sessions` 是进程内字典，没有持久化、跨实例共享或用户键 TTL。
- 当前没有请求级限流；媒体接口只有图片和视频大小限制，`.npz` 入口尚无总大小上限。
- `/health` 固定返回进程状态和版本，不探测任何模型、数据库或外部服务。
- `/chat` 的顶层异常仍可能把原始异常文本写入 HTTP 500 detail，日志也会记录部分用户输入。

## 文档调整

- 根 README 和项目事实 README 明确“本地原型，不可直接公网部署”。
- API 文档区分 liveness 与 readiness，并说明 `user_id` 不是身份凭证。
- RUNBOOK 在 `0.0.0.0` 联调命令前增加受信任网络警告和生产化检查清单。
- 面试问答增加安全部署高压问题，避免把 Docker 文件存在等同于可安全上线。

## 后续实现优先级

1. 认证授权、历史资源所有权与请求限流。
2. CORS 白名单、HTTPS/WSS 和统一安全错误响应。
3. 带 TTL 的共享会话存储、删除机制、日志脱敏与审计。
4. liveness/readiness 分离及依赖级健康状态。
