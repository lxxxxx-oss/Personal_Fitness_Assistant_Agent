# Diet 结构化画像校验

## 问题

Diet 原实现要求 LLM 输出 JSON，但把返回值作为未校验字符串直接写入状态，再拼接到检索 query 和推荐 Prompt。非法 JSON、越界数值或任意文本都可能污染后续链路。

## 修复

- 新增 `DietProfile` Pydantic 模型。
- 身高限制为 80–250cm，体重限制为 20–400kg；性别、目标使用受控枚举，偏好限制长度。
- `parse_diet_profile()` 从模型文本提取 JSON 对象，并处理未知数值、常见中文枚举和偏好列表。
- 解析或校验失败时使用全未知画像，并追加 `diet_profile_fallback:*` warning。
- 检索和推荐只使用 `model_dump()` 后的结构化对象。

## Tool Spec 检查

- 职责：画像模型只描述和校验 Diet 输入，不承担营养计算。
- 输入：一个 LLM 文本；只接受其中的 JSON 对象。
- 输出：稳定的 `DietProfile` 与可选安全错误原因。
- 权限：纯内存解析，不持久化用户身体信息。
- 错误：不返回原始 Pydantic 异常，使用可处理的 fallback warning。

## 边界

范围校验只能阻止明显异常值，不能证明用户输入真实，也不能替代营养师判断。生产化仍需用户确认、隐私授权、字段级删除和敏感数据存储治理。
