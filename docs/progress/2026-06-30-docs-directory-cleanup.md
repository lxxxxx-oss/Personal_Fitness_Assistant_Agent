# 2026-06-30 docs 目录收口整理

## 操作类型

文档结构调整 / 链接修复 / 历史记录收口。

## 背景

`docs/` 已形成 README、API、面试、技术、progress 和 tests 等分区，但顶层导航重复、README 运行命令过重，`progress/` 累积到 42 份文件，阅读成本高。

## 本次调整

- 将安装、启动、配置、测试和联调命令拆分到 `docs/RUNBOOK.md`。
- 精简 `docs/README.md`，只保留项目当前能力、边界、接口概览和下一步。
- 精简 `docs/DOCUMENTATION_MAP.md`，只保留阅读导航和文档分流规则。
- 修复小程序和历史技术文档中的失效相对链接。
- 将面试准备优化计划移动到 `docs/technical/interview-archive/`。
- 将 `docs/progress/` 调整为“索引 + 最近 10 条日期记录”，删除更早过程记录。
- 明确 `technical/interview-archive/` 只用于历史追溯，不再承担当前状态维护。

## 影响范围

只修改 `docs/`，未修改应用代码、配置、测试源码或接口行为。

## 验证

- 已检查 `docs/` 下 Markdown 本地链接，不存在失效链接。
- 已确认 `docs/progress/` 最终保留 10 条日期记录和 1 个索引。
- 当前状态、接口、运行、面试、技术设计、近期过程和测试证据已经具有独立入口。

## 后续维护

新增 progress 记录后删除最旧一条。删除前若其中包含仍然有效的长期结论，应先迁移到对应事实文档或专题文档。
