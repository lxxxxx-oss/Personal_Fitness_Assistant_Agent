# 文档学习入口

这个目录现在按“先学习、再查证据”的方式组织。你准备面试时，不需要从头翻完整个 `docs/`，按下面顺序看就够了。

## 先记住一句话

```text
面试资料用来背，项目总览/接口说明/运行排错用来查事实，
优化路线/技术细节用来解释设计，项目证据用来证明项目不是 toy。
```

## 目录怎么理解

| 层级 | 目录或文件 | 你要怎么用 |
|---|---|---|
| 学习区 | `interview/` | 面试直接背这里，按 P0/P1/P2 学 |
| 事实区 | `01_项目总览.md`、`02_接口说明.md`、`03_运行排错.md` | 查当前项目状态、接口和运行命令 |
| 设计区 | `optimization/`、`technical/`、`miniprogram/` | 被深入追问时查设计细节 |
| 证据区 | `04_项目证据.md` | 证明做过什么、测过什么，只看关键结论 |
| 历史区 | `superpowers/`、`technical/interview-archive/` | 早期材料，不作为日常学习入口 |

## 最近新增重点

上下文压缩和记忆系统优化后，当前最值得掌握的表达是：

> 项目实现了 Agent Memory & Context Engineering 闭环：把短期对话、压缩摘要和长期用户画像分层管理；长期记忆以 SQLite 为 source of truth，通过敏感信息候选确认、FTS5/LIKE 与 Milvus 语义召回、Prompt 预算注入进入回答；同时用 Memory + Context + RAG benchmark 验证链路稳定性。

对应阅读：

- [optimization/IMPLEMENTATION_SEQUENCE.md](./optimization/IMPLEMENTATION_SEQUENCE.md)：实现顺序和当前做到哪一步。
- [04_项目证据.md](./04_项目证据.md)：Memory + Context + RAG benchmark 的设计和真实测试结果。

## 不建议直接从这些目录开始看

- `superpowers/`：早期方案，不代表当前项目事实。
- 早期 progress/tests 流水记录已按面试优先原则裁剪，不再作为学习资料保留。

发生冲突时，优先级为：

```text
代码与自动化测试
  > 项目总览 / 接口说明 / 运行排错
  > interview / optimization / technical
  > 项目证据
  > superpowers
```
