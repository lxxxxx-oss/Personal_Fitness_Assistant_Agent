# 项目证据与验证结果

这份文档只保留面试最有价值的“做过什么、怎么证明”的证据。它替代原来的 `progress/` 和 `tests/` 日期流水目录。

## 当前测试结论

- 全量默认回归：`189 passed, 2 skipped, 2 warnings`。
- Memory + Context + RAG 专项 benchmark：`8/8 passed`。
- Memory/Context/RAG 相关核心回归集：`137 passed, 1 warning`。
- 两个 warning 分别来自 Starlette TestClient/httpx 兼容层弃用提示，以及当前 Windows 工作区 `.pytest_cache` 写入受限提示，不影响项目验收结论。

## 面试最该记住的证据

| 能力 | 证据 | 面试口径 |
|---|---|---|
| Router | 有绿色回归集和 challenge set | Router 不是靠感觉调 prompt，而是用样本集验证多意图、低置信和困难表达 |
| Milvus RAG | 完成真实链路效果评测 | 验证过 Collection、写入、检索、source 透传和 API 主链路；后续再扩大 Recall@K、MRR 和延迟基线 |
| Motion 图片/视频 | 真实 MediaPipe 链路可跑通 | 图片能提取单帧姿态，视频能通过 OpenCV 抽帧并生成多帧 PoseSequence |
| ToolRegistry | 有执行观测字段 | 工具调用保留 `execution_id`、耗时、attempts、fallback 归因和 audit log |
| Context Compression | 有 compact 可观测性 | 长 prompt 触发确定性 compact，最终 prompt 不超过上限，并公开 `_prompt_meta` 与 `execution` 轨迹 |
| Long-term Memory | 有候选确认和检索注入 | 健康/过敏等敏感记忆先进 candidate，确认后才进入长期记忆和 prompt |
| Milvus 用户记忆 | SQLite + Milvus 分层 | SQLite 是 source of truth，Milvus 只做可选语义召回增强，失败时主链路仍可用 |
| Memory + Context + RAG | 有统一 benchmark | 8 条联动样本覆盖长期记忆召回、敏感 candidate、prompt 注入、compact 和 RAG source 透传 |

## 怎么回答“你怎么证明项目不是 toy”

可以这样说：

> 我没有只做功能演示，而是给关键 Agent 能力补了可回归证据。Router 有离线 eval 和 challenge set；RAG/Milvus 做过真实链路验证；Motion 有真实图片和视频姿态链路；工具系统记录 execution_id、耗时、重试和 fallback；Memory + Context + RAG 还有统一 benchmark，验证长期记忆、上下文压缩和证据注入组合后没有回归。
