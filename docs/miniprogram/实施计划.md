# 微信小程序验收计划

早期按文件拆分的开发任务已经完成，旧版 `wx.request + enableChunked` SSE 方案也已退出当前代码，因此不再保留大段过期实现步骤。现行架构和完成度分别见[设计说明](./设计说明.md)与[实现状态](./实现状态.md)。

## 1. 当前目标

小程序代码已形成聊天、历史和 Motion 媒体上传的原型闭环。剩余工作不是继续开发页面，而是在微信开发者工具和真机中验证现有链路。

当前通信策略：

```text
WebSocket /chat/ws 作为 token 流式主路径
  -> 建连或执行失败
  -> POST /chat 普通 HTTP 降级
```

图片和视频分别通过 `/motion/analyze-image`、`/motion/analyze-video` 上传；标准动作列表来自 `/motion/references`。

## 2. 开发者工具验收

- 导入 `miniprogram/`，配置测试 AppID 和后端地址。
- 验证五类 intent、token 增量渲染、sources、warnings 和 execution 展示。
- 验证 WebSocket 失败后 HTTP 降级不会重复消息。
- 验证历史加载、刷新、清空和空状态。
- 验证图片 10 MB、视频 30 MB 限制以及 413/422/503 错误提示。
- 验证视频预览、上传进度、有效帧、FPS、置信度、warning 和相似度结果。

## 3. 真机验收

- 配置 HTTPS/WSS、`request`、`socket` 和 `uploadFile` 合法域名。
- 检查相册、相机、视频相关隐私声明和授权流程。
- 验证弱网、断网、后台切换、长回复滚动和大文件上传体验。
- 确认无兼容标准动作时安全退回姿态提取，不伪造相似度。

## 4. 完成标准

只有开发者工具和真机核心链路都通过，才能把“小程序已实现”升级为“移动端已完成端到端验收”。在此之前，面试中统一表述为：

> 小程序代码级闭环已经完成，采用 WebSocket 流式主路径和 HTTP 降级，也接入了 Motion 图片/视频上传；当前边界是开发者工具、真机、HTTPS 和弱网还没有完成完整验收。
